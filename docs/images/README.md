# Merlin Documentation Screenshots

Wireframe placeholders ship with the repo so [Technician Quick Start](../Technician-Quick-Start.md) renders in GitHub and print pipelines. **Replace with dealership-specific PNG captures** before laminating bay cards.

## Required images

| Filename | Capture |
|----------|---------|
| `technician-login-ro-list.svg` | Login screen and repair order list |
| `technician-voice-modes.svg` | Tap-to-toggle vs push-to-talk controls |
| `technician-voice-panel.svg` | Listening panel with noise meter and confidence |
| `technician-notes-voice.svg` | Technician notes field with microphone button |
| `technician-diagnostic-evidence.svg` | Diagnostic Evidence section with photo queue |
| `technician-generate-mi43.svg` | **Generate MI 4.3** primary button |
| `technician-story-actions.svg` | Story editor with Audit Story, Review with AI, Copy for CDK |

PNG or WebP with the same base name also satisfy pre-rollout validation.

## Capture tips

- Use a shop-floor tablet at 480px width (Merlin mobile shell)
- Chrome or Edge, signed in as technician
- Blur or redact real customer VINs and names
- PNG format at 2× resolution for print clarity

Pre-rollout validation warns if images are missing — this does not block launch but is required before print distribution.
